# ЕГРН-Парсер — инструкция для пользователя

Приложение формирует отчёты по открытым сведениям ЕГРН через API **Контур.Реестро**:
для каждого объекта с кадастровым номером (КН) создаётся **PDF** и строки в сводном **Excel-отчёте**.

> **Пошаговый разбор экрана со скриншотами:** [`docs/ПОШАГОВАЯ_ИНСТРУКЦИЯ.md`](docs/ПОШАГОВАЯ_ИНСТРУКЦИЯ.md)

> **Подробно: форма собственности и обременения (Росреестр):** [`docs/ИНСТРУКЦИЯ_СОБСТВЕННИКИ.md`](docs/ИНСТРУКЦИЯ_СОБСТВЕННИКИ.md)

---

## 1. Что нужно для работы

| Требование | Пояснение |
|------------|-----------|
| **Windows 10/11** | 64-bit |
| **Интернет** | исходящий HTTPS на `api.kontur.ru:443` |
| **Ключ API** | `apiKey` и `orgId` из личного кабинета Контур.Реестро |
| **Входной файл** | Excel или CSV со столбцом **ЕГРН** (кадастровый номер) |
| **Python не нужен** | всё уже внутри приложения |

**Не требуется:** установка Python, pip, командная строка.

---

## 2. Установка

> **Важно:** программа состоит из **папки целиком** (`ЕГРН-Парсер.exe` + папка `_internal`).
> Если запустить `.exe` без `_internal` или из каталога `build\egrn\`, появится ошибка:
> *«Failed to load Python DLL python311.dll»*.

### Вариант А — установщик (рекомендуется)

1. Сначала на машине разработчика должна быть **полная сборка** в `dist\ЕГРН-Парсер\`
   (см. `build\build_exe.bat`), затем — компиляция `build\installer.iss` в Inno Setup.
2. Запустите **`ЕГРН-Парсер_Setup_0.1.0.exe`** из папки **`reestro_app\dist_installer\`**
   (после компиляции `build\installer.iss` в Inno Setup).
3. Следуйте шагам мастера (можно оставить путь по умолчанию).
4. При желании отметьте **«Создать ярлык на рабочем столе»**.
5. После установки запустите **ЕГРН-Парсер** из меню Пуск или с рабочего стола.

### Вариант Б — папка без установки (portable)

1. Скопируйте **всю папку** `dist\ЕГРН-Парсер\` на компьютер (например, `C:\Реестро\ЕГРН-Парсер\`).
2. Внутри должны быть:
   - `ЕГРН-Парсер.exe`
   - папка **`_internal`** (десятки файлов, в т.ч. `python311.dll`)
   - `ПРОЧТИ_МЕНЯ.txt`, `ИНСТРУКЦИЯ.md`
3. Запустите **`ЕГРН-Парсер.exe`** из этой папки.

### ❌ Нельзя так

| Неправильно | Почему |
|-------------|--------|
| Запуск `build\egrn\ЕГРН-Парсер.exe` | Это **промежуточный** файл сборки, рядом нет `_internal` |
| Скопировать только один `.exe` | Без `_internal` программа не запустится |
| Удалить или переименовать `_internal` | Там все библиотеки Python и Qt |

> Настройки (`config.json`) сохраняются **рядом с `.exe`**, в той же папке, где лежит программа.

## 3. Первый запуск — настройка подключения

Откройте вкладку **«Подключение»**.

| Поле | Что ввести |
|------|------------|
| **baseUrl** | Обычно `https://api.kontur.ru` (не менять без указания IT) |
| **apiKey** | Ключ API (формат UUID, напр. `91e4ea5e-35bd-...`) |
| **orgId** | ID организации (формат UUID) |
| **proxy** | Только если IT выдал корпоративный прокси, напр. `http://proxy.company.ru:8080` |

**Параметры сети** (можно оставить по умолчанию):

| Параметр | Значение по умолчанию | Зачем |
|----------|----------------------|-------|
| Таймаут | 120 сек | ждать ответ сервера дольше при медленной сети |
| Повторов при обрыве | 5 | автоматические повторы при обрыве соединения |
| Пауза между запросами | 2 сек | снижает нагрузку на API и файрвол |
| Автосохранение | каждые 5 объектов | report не теряется при сбое |

**Порядок действий:**

1. Введите **apiKey** и **orgId**.
2. Нажмите **«Проверить подключение»**.
   - Успех: зелёное сообщение и остаток единиц `address_api_open_data`.
   - Ошибка 401 — неверный ключ или orgId.
   - Ошибка 402 / «нет единиц» — пополните баланс в Контур.Реестро.
   - «Нет связи» — проблема сети или файрвол (см. раздел 10).
3. Нажмите **«Сохранить»**.

## 4. Структура результатов

Все результаты складываются в **одну папку результатов** (вы выбираете её на вкладке «Пакетная обработка»).

```
C:\Реестро\output\
├── report.xlsx          ← сводная таблица (или имя, заданное вручную)
├── pdf\                 ← один PDF на каждый объект с КН
│   └── {uuid}.pdf
├── cache\
│   ├── json\            ← ответы API (для повторов без запросов)
│   └── kn_index.json    ← защита от дублей PDF
└── logs\
    └── run_YYYYMMDD_HHMMSS.jsonl   ← журнал прогона
```

**Важно:** всегда используйте **одну и ту же папку** для одного проекта. Программа запоминает уже обработанные объекты и не запрашивает их повторно.

### Куда и под каким именем сохраняется Excel-отчёт

| Что | Где задаётся | Куда попадает |
|-----|--------------|---------------|
| **Папка** | «Пакетная обработка» → **Папка результатов** | Корень output, напр. `C:\Реестро\output\` |
| **Имя файла** | «Пакетная обработка» → **Имя файла отчёта** | Файл `.xlsx` прямо в корне output |
| **PDF** | автоматически | Подпапка `output\pdf\` |

- По умолчанию включено **«Авто-имя»** — отчёт называется `Отчёт_Запрос_20260620.xlsx` (имя входного файла + дата).
- Снимите **«Авто-имя»** и введите своё имя, напр. `Сводка_участки.xlsx`.
- Имя **`report.xlsx`** — историческое из ТЗ/образца; используется только если авто-имя выключено и поле пустое.
- **Не выбирайте** подпапку `pdf\` как папку результатов — иначе PDF окажутся в `pdf\pdf\`, а отчёт не там, где ожидаете.

## 5. Вкладки приложения — краткий обзор

| Вкладка | Назначение |
|---------|------------|
| **Подключение** | Ключ API, проверка баланса, параметры сети, логин Госуслуг |
| **Пакетная обработка** | Основная работа: таблица → PDF + Excel |
| **Одиночный объект** | Один или несколько КН вручную |

| **Анализ логов** | Просмотр ошибок, повтор проблемных объектов |
| **Результаты** | Сводка последнего прогона, открыть отчёт и папку |

В правом нижнем углу окна — подпись **by BeRealBear**.

## 6. Сценарии работы

### Сценарий 1. Первый полный прогон по таблице

**Цель:** обработать все объекты из файла `Запрос.xlsx`.

1. **Подключение** → проверить API → **Сохранить**.
2. **Пакетная обработка**:
   - **Входная таблица** → **Обзор…** → выберите `Запрос.xlsx`.
   - Под полем появится: `строк: N   с кадастровым номером: M`.
   - **Папка результатов** → **Обзор…** → выберите или создайте, напр. `C:\Реестро\output`.
   - **Авто-имя** — включено: отчёт будет вида `Отчёт_Запрос_20260619.xlsx`.
   - **Диапазон** — оставьте «с начала» / «до конца».
   - **Лимит** — «без лимита» (или задайте число, если нужна пробная партия).
3. Нажмите **«Старт»**.
4. Следите за прогрессом и счётчиками:
   - **OK** — объект обработан, PDF создан;
   - **Без сведений** — API ответил, но данных нет (404 и т.п.);
   - **Пропущено** — уже было в report ранее;
   - **Обрыв сети** — объект **не записан**, повторится при следующем запуске;
   - **Без КН** — в строке таблицы нет номера, PDF не создаётся.
5. По завершении откройте вкладку **«Результаты»** → **Открыть report.xlsx** или **Открыть папку результатов**.

> Перед запуском **закройте `report.xlsx` в Excel**, иначе сохранение может не пройти.

---

### Сценарий 2. Продолжить после остановки или обрыва сети

**Цель:** догнать необработанные объекты, не тратя API на готовые.

1. **Не меняйте папку результатов** — та же `output`, что и раньше.
2. **Пакетная обработка** → та же входная таблица → **Старт**.
3. Готовые объекты будут **пропущены** (счётчик «Пропущено»).
4. Объекты с **обрывом сети** из прошлого прогона будут запрошены снова.

**Альтернатива — через логи:**

1. **Анализ логов** → выберите последний лог `run_...jsonl`.
2. Отфильтруйте статус **«Обрыв сети»** или **«Без сведений»**.
3. Нажмите **«Повторить проблемные (сеть/ошибки)»** → подтвердите.
4. Программа обработает только проблемные КН в ту же папку.

---

### Сценарий 3. Обработать диапазон объектов (например, с 30 по 75)

**Цель:** не весь файл, а только часть строк.

1. **Пакетная обработка** → выберите таблицу и папку результатов.
2. **Диапазон объектов:**
   - **с №** → `30`
   - **по №** → `75`
3. Нумерация — **по строкам входного файла** (как в Excel, с первой строки данных).
4. **Старт**.

Объекты вне диапазона не обрабатываются. Уже готовые внутри диапазона всё равно **пропускаются**.

---

### Сценарий 4. Обработать партию (лимит) — по 20–50 за раз

**Цель:** не тратить весь баланс API за один раз или снизить риск при нестабильной сети.

1. **Пакетная обработка** → таблица и папка.
2. **Лимит новых объектов** → напр. `20` (считаются только **новые** запросы; пропущенные в лимит не входят).
3. **Старт** → дождитесь окончания.
4. Повторяйте **Старт** с той же папкой, пока «Пропущено» не станет равно числу объектов с КН.

---

### Сценарий 5. Один или несколько объектов по КН

**Цель:** проверить один проблемный объект или быстро добавить 2–3 КН.

1. Вкладка **«Одиночный объект»**.
2. Введите КН **по одному в строке**, формат: `43:40:000864:351`.
3. **Папка результатов** — та же, что для основного проекта (чтобы report дополнялся).
4. **Обработать**.
5. **Открыть папку PDF** — найти свежий файл.

Одиночная обработка **перезаписывает** данные по этому КН в report (режим force).

---

### Сценарий 6. Report «разъехался» или много лишних PDF

**Цель:** привести report и PDF в порядок без новых запросов к API.

1. Вкладка **«Обслуживание»**.
2. Укажите **папку результатов** — **корень output** (где лежат `report.xlsx`, папки `pdf\` и `cache\json\`).
   - **Не выбирайте** подпапку `pdf\` или `cache\json\` — программа подставит корень сама, но надёжнее указать сразу `output` или `output_test_first`.
   - Под полем пути появится строка вида **«JSON-кэш: N файлов»** — если 0, проверьте путь (на скрине часто путают `output` и `output_test_first`).

**Шаг А — анализ (без изменений):**

3. **«Анализ дублей (пробно)»** — покажет, сколько лишних PDF и дублей КН.

**Шаг Б — очистка:**

4. **«Очистить дубли (применить)»** → подтвердите.
   - Лишние PDF переносятся в **`pdf_old`** (если включена галочка), не удаляются навсегда.
   - В report остаётся один блок строк на каждый КН.

**Шаг В — пересборка report из кэша** (если report потерял строки, а JSON-кэш есть):

5. **«Пересобрать из кэша»** — создаёт `report.xlsx` заново по всем файлам в `cache\json\` **без API**.
6. Галочка **«При пересборке не пересоздавать PDF»** — оставьте включённой, если PDF уже есть.

---

### Сценарий 7. Анализ логов и поиск ошибок

**Цель:** понять, что пошло не так, и повторить только проблемные объекты.

1. Вкладка **«Анализ логов»**.
2. Выберите лог из списка (`run_YYYYMMDD_HHMMSS.jsonl`) или **«Папка результатов…»**.
3. Вверху — **сводка**: OK, обрыв сети, ошибки, длительность.
4. **Фильтры:**
   - уровень: INFO / WARN / ERROR;
   - статус: OK, обрыв сети, без сведений и т.д.;
   - **поиск** по КН или тексту.
5. **Двойной клик** по строке — КН копируется в буфер.
6. **«Экспорт в CSV»** — отправить лог IT или исполнителю.
7. **«Повторить проблемные»** — автоматически обработать объекты с обрывом сети / ошибками из этого лога.

---

### Сценарий 8. Собственники: форма собственности и обременения (Росреестр)

**Зачем:** API Контур.Реестро отдаёт только вид и дату права. **Форму собственности**
(Частная / Муниципальная / Общая долевая / Государственная) и **обременения**
(ипотека, аренда, арест и т.п.) можно взять только с **lk.rosreestr.ru**.

> Открытые сведения ЕГРН **не содержат ФИО** физлиц — по закону. В колонке
> «Вид собственности» и в поле PDF «Правообладатель» указывается **форма
> собственности**, а не имя человека.

**Что нужно:** доступ кадастрового инженера к Госуслугам, российский IP, интернет.
Браузер **уже встроен в программу** — ничего скачивать и устанавливать не нужно.
(Если по какой-то причине встроенного браузера нет, он скачается автоматически
один раз в папку `ms-playwright` рядом с программой.)

**Порядок (сбор автоматический):**

1. Вкладка **«Подключение»** → внизу укажите **логин Госуслуг** (для удобства;
   пароль и код подтверждения вводятся в браузере и нигде не сохраняются).
2. Вкладка **«Собственники (Росреестр)»** → выберите **папку результатов** (тот же
   `output`, что и в обработке). Список КН берётся из её `report.xlsx`.
3. Нажмите **«Запустить автоматический сбор»**. Откроется браузер:
   - если требуется — войдите через **Госуслуги** (кадастровым инженером), введите
     код подтверждения и нажмите в программе **«Вход выполнен — продолжить»**. Это
     нужно **один раз** — сессия сохранится для следующих запусков;
   - дальше программа **сама** по каждому КН открывает карточку, считывает форму
     собственности и обременения и сохраняет в кэш. Прогресс виден в полосе.
4. По галочке **«После сбора сразу применить…»** данные автоматически попадут в
   `report.xlsx` (колонка «Вид собственности») и в PDF (поле «Правообладатель» и
   «Ограничение прав и обременение»). Запросы к API при этом **не тратятся**.

Кнопка **«Применить из кэша (Пересобрать)»** делает то же самое вручную, без
браузера — пригодится, если данные уже собраны.

> **Почему нужен разовый вход.** Госуслуги требуют код подтверждения (2FA) — его
> нельзя ввести за человека. Поэтому пароль в программе **не хранится**: вы вводите
> его в браузере сами один раз. Благодаря сохранённой сессии повторно логиниться,
> как правило, не нужно — последующие сборы идут полностью без участия оператора.
> Если у кадастрового инженера не появляется капча — пачка собирается целиком сама.

## 7. Как читать report.xlsx

| Что | Пояснение |
|-----|-----------|
| **Sheet1** | Данные — открывайте эту вкладку |
| **Сводка** | Статистика последнего прогона (служебная) |
| **Строк** может быть больше, чем **объектов** | Одна строка = **одно право**; у объекта может быть 2–5 строк |
| **Уникальных КН** | Считайте уникальные значения в колонке **«Кадастровый номер»** |
| Колонка **«Отчёт»** | Имя PDF-файла для этого объекта |

## 8. Правила, которые важно соблюдать

1. **Одна папка результатов** на один проект — не создавайте новую папку для каждого запуска.
2. **Закрывайте report.xlsx в Excel** во время работы программы.
3. **Не удаляйте** папку `cache\json\` — это экономит платные запросы к API.
4. **Обрыв сети** — не страшно: объект не записывается, при следующем **Старт** или **Повторить проблемные** он обработается снова.
5. **Строки без КН** попадают только в Excel, PDF по ним **не создаётся** — это норма.
6. **Баланс API** — каждый **новый** объект с данными в ЕГРН списывает 1 единицу `address_api_open_data`. Проверяйте на вкладке «Подключение».

---

## 9. Кнопки «Старт» и «Стоп»

| Кнопка | Действие |
|--------|----------|
| **Старт** | Начать обработку |
| **Стоп** | Остановить **после текущего объекта** (не мгновенно). Уже обработанное сохранено благодаря автосохранению |

Во время работы другие вкладки доступны; второй прогон параллельно **нельзя** запустить.

## 10. Типичные проблемы

| Симптом | Что делать |
|---------|------------|
| **401 — неверный ключ** | Проверьте apiKey и orgId на «Подключении» |
| **402 / нет единиц** | Пополните баланс в Контур.Реестро |
| **Обрыв сети / таймаут** | Увеличьте таймаут и паузу на «Подключении»; повторите через «Анализ логов» |
| **report не обновляется** | Закройте Excel с report.xlsx |
| **Много PDF, мало строк в report** | «Обслуживание» → «Пересобрать из кэша» или «Очистить дубли» |
| **Failed to execute script 'main' / relative import** | Старая сборка. Пересоберите: `build\build_exe.bat`, затем installer.iss заново |
| **Failed to load Python DLL python311.dll** | Запуск не из той папки. Нужна **`dist\ЕГРН-Парсер\`** с папкой `_internal`, не `build\egrn\`. Пересоберите: `build\build_exe.bat` |
| **Файрвол блокирует** | Попросите IT открыть исходящий HTTPS на `api.kontur.ru:443` |

## 11. Быстрая шпаргалка

```
Первый раз:
  Подключение → apiKey/orgId → Проверить → Сохранить
  Пакетная обработка → таблица + папка output → Старт

Продолжить:
  Та же папка output → Старт (готовые пропустятся)

Часть файла (30–75):
  Пакетная → диапазон с 30 по 75 → Старт

Один КН:
  Одиночный объект → ввести КН → Обработать

После обрыва сети:
  Анализ логов → Повторить проблемные

Порядок в report/PDF:
  Обслуживание → Анализ дублей → Очистить / Пересобрать из кэша
```
