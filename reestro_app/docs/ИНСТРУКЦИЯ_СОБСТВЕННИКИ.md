# Собственники и обременения — подробная инструкция

Эта инструкция описывает **новый функционал**: автоматическое получение **формы собственности** и **обременений** с портала Росреестра и внесение их в **Excel** и **PDF**.

> **Кратко:** сначала обрабатываете объекты через Контур (как раньше), затем на вкладке **«Собственники (Росреестр)»** один раз входите через Госуслуги — дальше программа сама собирает данные и обновляет отчёты.

---

## 1. Что получится на выходе

После полного цикла в отчётах появятся данные, как на образцах заказчика:

| Где | Поле | Пример значения |
|-----|------|-----------------|
| **Excel**, колонка **«ФИО»** | форма собственности (с Росreestr) | `Частная собственность`, `Муниципальная собственность` |
| **Excel**, колонка **«Вид собственности»** | тип зарегистрированного права (из API Контура) | `Собственность`, `Аренда` |
| **PDF**, раздел 2, **«Правообладатель»** | форма собственности | `Частная собственность` |
| **PDF**, раздел 2, **«Ограничение прав и обременение объекта недвижимости»** | обременения | `№ … от … (Ипотека в силу закона)` или `не зарегистрировано` |

### Чего программа **не может** выдать

По **открытым сведениям ЕГРН** (и по закону) **ФИО физических лиц не публикуются**.
Колонка Excel **«ФИО»** заполняется **формой собственности** с Росreestr (не именем человека).
Если форма ещё не собрана — там может быть «данные отсутствуют».

---

## 2. Что нужно заранее

| Требование | Обязательно | Пояснение |
|------------|-------------|-----------|
| Windows 10/11 (64-bit) | да | Программа уже собрана, Python не нужен |
| Интернет | да | API Контура + сайт Росреестра |
| **Российский IP** | да для Росреестра | С зарубежного VPN `lk.rosreestr.ru` часто не открывается |
| Ключ **Контур.Реестро** (apiKey, orgId) | да | Для первичной загрузки объектов |
| Учётная запись **кадастрового инженера** на Госуслугах | да | Для входа на Росреестр (один раз) |
| Телефон для **кода подтверждения** Госуслуг | да, один раз | SMS/код вводится в браузере вручную |
| Браузер Chromium | **уже внутри программы** | Ничего скачивать не нужно |

**Пароль Госуслуг в программу не вводится** — только в открывшемся браузере. Так безопаснее, и Госуслуги всё равно требуют код подтверждения.

## 3. Полный порядок работы (с нуля)

### Шаг 0. Распаковка

1. Распакуйте архив **целиком** в папку, например `C:\ЕГРН\`.
2. Внутри должны быть:
   - `ЕГРН-Парсер.exe`
   - папка `_internal` (не удалять!)
   - `ПРОЧТИ_МЕНЯ.txt`, `ИНСТРУКЦИЯ.md`, папка `docs\`
3. Запустите **`ЕГРН-Парсер.exe`**.

> ❌ Нельзя копировать только один `.exe` без `_internal` — программа не запустится.

---

### Шаг 1. Подключение к Контур.Реестро

1. Откройте вкладку **«Подключение»**.
2. Введите **apiKey** и **orgId** из личного кабинета Контур.Реестро.
3. Нажмите **«Проверить подключение»** — должно быть зелёное сообщение и остаток единиц.
4. Нажмите **«Сохранить»**.

**Блок «Госуслуги / Росреестр»** (внизу той же вкладки):

- Укажите **логин Госуслуг** кадастрового инженера (email или телефон) — **для удобства**, чтобы не забыть, под кем входить.
- Пароль здесь **не вводится**.
- Нажмите **«Сохранить»** ещё раз.

---

### Шаг 2. Пакетная обработка (как раньше)

1. Вкладка **«Пакетная обработка»**.
2. **Входной файл** — Excel/CSV со столбцом кадастровых номеров (ЕГРН).
3. **Папка результатов** — выберите **корень** output, например `C:\ЕГРН\output\`.
   - ✅ Правильно: `C:\ЕГРН\output\`
   - ❌ Неправильно: `C:\ЕГРН\output\pdf\` (это подпапка!)
4. При необходимости задайте имя отчёта или оставьте **«Авто-имя»**.
5. Нажмите **«Старт»** и дождитесь окончания.

**Результат шага 2:**

```
C:\ЕГРН\output\
├── report.xlsx          ← или ваше имя, напр. Отчёт_Запрос_20260621.xlsx
├── pdf\                 ← PDF по каждому объекту
├── cache\json\          ← кэш API (нужен для пересборки без оплаты)
└── logs\
```

На этом этапе в Excel/PDF уже есть адрес, площадь, вид права, дата регистрации и т.д., но **«Вид собственности»** может быть «данные отсутствуют» — это нормально до шага 3.

---

### Шаг 3. Сбор формы собственности и обременений (Росреестр)

1. Откройте вкладку **«Собственники (Росреестр)»**.

2. **Папка результатов** — укажите **ту же папку**, что в шаге 2 (`C:\ЕГРН\output\`).
   - Программа сама найдёт корень, даже если вы случайно указали подпапку.
   - Под полем появится строка: сколько объектов в кэше API и сколько уже собрано с Росреестра.

3. Настройки (галочки):

   | Галочка | Рекомендация | Зачем |
   |---------|--------------|-------|
   | **Только объекты с зарегистрированными правами** | ✅ включена | Без прав форма собственности бессмысленна |
   | **Пересобрать уже собранные** | выключена | Включите, если нужно обновить данные по всем КН заново |
   | **После сбора сразу применить к report.xlsx и PDF** | ✅ включена | После **каждого** успешно собранного объекта обновляет Excel и PDF |
   | **Проверочный режим (с оператором в браузере)** | выключена | КН подставляется автоматически; вы выбираете «Вид объекта» и нажимаете «Найти» |

4. Нажмите **«Запустить автоматический сбор»** (или проверочный сбор, если включена галочка выше).

5. **Откроется окно браузера** (Microsoft Edge или Google Chrome — что установлено в Windows). Возможны два варианта:

   **А) Вы уже входили раньше на этом компьютере** — браузер откроет «Справочную информацию online», кнопка **«Вход выполнен — продолжить»** станет активной — нажмите её.

   **Б) Первый вход** — выполните в браузере:
   - нажмите вход через **Госуслуги**;
   - введите логин и пароль кадастрового инженера;
   - введите **код подтверждения** из SMS/приложения Госуслуг;
   - выберите пользователя, если спросит;
   - программа сама перейдёт на **«Справочная информация по объектам online»** (не «Мои заявки»);
   - когда кнопка **«Вход выполнен — продолжить»** станет активной — нажмите её.

6. **Автоматический режим** (галочка «Проверочный режим» выключена):
   - программа по очереди берёт кадастровые номера из `report.xlsx`;
   - на сайте Росreestr выбирает **«Вид объекта»** из выпадающего списка (кликом, не вводом текста);
   - вводит КН, нажимает **«Найти»**, открывает справку;
   - считывает **форму собственности** и **обременения**;
   - сохраняет в `cache\rosreestr\{номер}.json`;
   - при включённой галочке «После сбора сразу применить…» — сразу обновляет Excel и PDF;
   - показывает прогресс в полосе и в журнале внизу вкладки.

   **Проверочный режим** (галочка включена):
   - для каждого КН программа **автоматически подставляет номер** в поле «Адрес или кадастровый номер»;
   - показывает подсказку по **виду объекта** (из кэша API);
   - **вы в браузере**: выберите «Вид объекта» → **«Найти»** → откройте справку;
   - нажмите **«Данные готовы — записать и далее»** — программа прочитает страницу, сохранит и перейдёт к следующему объекту.

7. Управление во время сбора:

   | Кнопка | Действие |
   |--------|----------|
   | **Остановить** | Немедленно прервать сбор (уже сохранённое останется) |
   | **Пропустить объект** | Перейти к следующему КН, если текущий завис или не находится |
   | **Сбросить профиль браузера** | Если белый экран или ERR_CONNECTION_RESET — сброс сессии (повторный вход в Госуслуги) |

8. По окончании откройте результат:
   - вкладка **«Результаты»** → **«Открыть отчёт»**;
   - или вручную: папка output → `report.xlsx` и `pdf\`.

---

### Шаг 4. Повторные запуски (тот же проект)

Если добавили новые объекты в пакетную обработку:

1. Снова **«Пакетная обработка»** — обработайте только новые (старые пропустятся по кэшу).
2. **«Собственники (Росреестр)»** → **«Запустить автоматический сбор»**.
   - Повторный вход через Госуслуги **обычно не нужен** — сессия сохранена в профиле браузера.
   - Соберутся только **новые** КН (если не включена «Пересобрать уже собранные»).

## 4. Кнопки вкладки «Собственники (Росреестр)»

| Кнопка | Действие |
|--------|----------|
| **Обзор…** | Выбор папки результатов (корень output) |
| **Запустить автоматический сбор** | Браузер + сбор по всем нужным КН |
| **Вход выполнен — продолжить** | Подтверждение входа (активна, когда форма поиска готова) |
| **Остановить** | Немедленно прервать сбор |
| **Пропустить объект** | Следующий КН без ожидания текущего |
| **Данные готовы — записать и далее** | В проверочном режиме: записать данные с открытой справки |
| **Сбросить профиль браузера** | Сброс сессии браузера (при белом экране / сбросе соединения) |
| **Применить из кэша (Пересобрать)** | Обновить Excel/PDF из уже собранного кэша **без браузера** |

## 5. Где лежат собранные данные

```
output\
├── cache\
│   ├── json\              ← ответы API Контура
│   └── rosreestr\         ← форма собственности + обременения с Росреестра
│       ├── 43_40_000864_555.json
│       └── ...
├── logs\
│   ├── run_20260621_153045.jsonl       ← лог пакетной обработки
│   └── rosreestr_20260621_160012.jsonl ← лог сбора собственников
├── report.xlsx
└── pdf\
```

**Журнал сбора:** все строки с вкладки «Собственники» пишутся в  
`output\logs\rosreestr_ГГГГММДД_ЧЧММСС.jsonl`  
(в конце сбора в окне будет «Лог записан: …»).  
Смотреть: вкладка **«Анализ логов»** → папка output → файл `rosreestr_…`.

Пример содержимого `cache\rosreestr\*.json`:

```json
{
  "cadastralNumber": "43:40:000864:555",
  "ownership_form": "Частная собственность",
  "encumbrances": [
    "№ 43-43-01/024/2014-742 от 14.04.2014 (Ипотека в силу закона)"
  ]
}
```

## 6. Типичные проблемы

| Симптом | Причина | Что делать |
|---------|---------|------------|
| Браузер снова просит логин каждый раз | Сессия сброшена / другой пользователь Windows | Войдите через Госуслуги ещё раз, нажмите «Вход выполнен — продолжить» |
| Сайт Росреестра не открывается | Зарубежный VPN / блокировка | Отключите VPN или используйте российский IP |
| «Нет кэша API» | Не было пакетной обработки | Сначала шаг 2 — «Пакетная обработка» |
| «Нет КН для сбора» | Всё уже в `cache\rosreestr` | Включите «Пересобрать уже собранные» или добавьте новые объекты |
| Форма не распознана для части КН | Неверный «Вид объекта» или объект не найден | Включите **проверочный режим** и выберите вид вручную; проверьте КН |
| В журнале «не удалось выбрать … в выпадающем списке» | Автовыбор вида объекта не сработал | **Проверочный режим** или выберите вид вручную в браузере |
| В Excel «ФИО» пустой | Не собраны данные с Росreestr | Запустите сбор; нажмите **«Применить из кэша»** |
| Программа «зависла» на сборе | Долгая загрузка / лимит Росreestr | **Пропустить объект** или **Остановить**; при лимите подождите ~90 с |
| Белый экран / ERR_CONNECTION_RESET | Профиль браузера повреждён | **Сбросить профиль браузера** → войти в Госуслуги заново |

---

## 7. Безопасность

- **Пароль Госуслуг не сохраняется** в `config.json` и нигде на диске программой.
- Сессия браузера хранится в служебном профиле рядом с программой (`browser_profile_rosreestr`).
- **Не передавайте** папку с программой третьим лицам вместе с активной сессией инженера.
- На общем компьютере после работы можно удалить папку профиля браузера (программа создаст новую при следующем входе).

---

## 8. Краткая шпаргалка (одна страница)

```
1. Распаковать архив → запустить ЕГРН-Парсер.exe
2. Подключение → apiKey, orgId → Проверить → Сохранить
3. Пакетная обработка → файл + папка output → Старт
4. Собственники (Росреестр) → та же папка output
5. Запустить сбор → войти через Госуслуги (1 раз) → Продолжить
   (если автоматика не находит объект — включить «Проверочный режим»)
6. Дождаться конца → открыть report.xlsx и pdf\
```

**Готово.** Excel: колонка **«ФИО»** = форма собственности. PDF: «Правообладатель» и «Обременение».
