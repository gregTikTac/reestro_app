# ЕГРН-Парсер — десктоп-приложение (Windows, .exe)

Графическая оболочка над существующим движком `reestro/reestro_parser.py`.
Цель: обычный сотрудник работает мышкой, без командной строки; ошибки сети и
внештатные ситуации отлавливаются и показываются понятно; дубли PDF и строк
Excel исключены на уровне логики.

---

## 1. Принцип: разделение «движок» и «интерфейс»

```
┌──────────────────────────────┐      ┌──────────────────────────────┐
│        GUI (PySide6)         │      │   Движок (engine, headless)   │
│  вкладки, кнопки, прогресс   │◄────►│  reestro_parser + обёртки      │
│  не знает про API/Excel      │ events│  API, Excel, PDF, кэш, дедуп   │
└──────────────────────────────┘      └──────────────────────────────┘
        ▲ поток UI                            ▲ рабочий поток (QThread)
```

- **Движок** — это текущий `reestro_parser.py` плюс тонкая обёртка `engine/runner.py`,
  которая вызывает обработку и шлёт **события** (прогресс, лог, ошибка, итог)
  через колбэки. Никаких `print` в GUI — только структурированные события.
- **GUI** запускает движок в **отдельном потоке** (QThread), чтобы окно не зависало,
  и умеет **Пауза / Стоп** (кооперативная отмена между объектами).

> Почему так: один и тот же движок работает и из .exe, и из консоли (для отладки),
> логика парсинга не дублируется, тестировать проще.

## 2. Стек и почему он

| Слой | Выбор | Причина |
|------|-------|---------|
| GUI | **PySide6 (Qt)** | Нативный вид, таблицы, прогресс, потоки «из коробки», стабильная сборка в .exe |
| Движок | существующий Python-код | Уже отлажен на боевых данных |
| Сборка .exe | **PyInstaller** (one-folder) + **Inno Setup** (установщик) | Все зависимости вшиваются в сборку; у пользователя НЕ нужен Python/pip |
| Хранение настроек | `config.json` + `settings.json` рядом с .exe | Простой перенос, ручное редактирование при желании |

Альтернатива GUI — CustomTkinter (легче, exe меньше ~30 МБ), но беднее по таблицам и
диалогам. Рекомендую **PySide6**; если важен минимальный размер — переключимся.

«Лёгкая установка»: пользователь получает **один установщик `Setup.exe`** (или папку с
`ЕГРН-Парсер.exe`). Зависимости уже внутри — интернет и pip при установке не нужны.

## 3. Вкладки приложения

### 3.1. «Подключение» (настройки API)
- Поля: `apiKey`, `orgId`, `baseUrl`, `proxy` (необязательно).
- Параметры сети по умолчанию: `timeout`, `retries`, `pause`, `save-every`.
- Кнопка **«Проверить подключение»** → запрос баланса, показывает: статус, остаток
  единиц `address_api_open_data`, понятную ошибку (401 ключ, 402 баланс, нет сети).
- Кнопка **«Сохранить»** → пишет `config.json`. apiKey можно маскировать (показать/скрыть).

### 3.2. «Пакетная обработка» (главная)
- **Входная таблица**: выбор `.xlsx/.csv` (кнопка + drag&drop), автоопределение
  колонки с КН, показ «найдено объектов с КН: N / всего строк: M».
- **Куда писать**: папка результатов + имя файла отчёта.
  - Имя **авто** по шаблону (напр. `Отчёт_{вх.файл}_{дата}.xlsx`) — переключатель,
  - либо **вручную**.
- **Диапазон**: «с объекта № __ по № __» (напр. 30–75). Пусто = все.
- **Параметры запуска**: лимит новых, пауза, таймаут, ретраи, save-every — с
  разумными значениями из вкладки «Подключение».
- **Кнопки**: Старт / Пауза / Стоп.
- **Прогресс**: полоса, «обрабатывается: КН … (45/521)», счётчики
  OK / без сведений / пропущено / **обрыв сети**.
- **Лог** внизу (цветной: ошибки красным, сеть жёлтым).

### 3.3. «Одиночный объект»
- Поле ввода одного или нескольких КН (по строке).
- Кнопка **«Обработать»** → сразу запрос, результат в таблице, кнопка «Открыть PDF».
- Удобно для точечной перепроверки проблемных объектов.

### 3.4. «Обслуживание»
- **Очистка дублей** (`cleanup_output`): сначала «пробный прогон» (сколько лишних
  PDF / дублей КН), затем **Применить** (с переносом лишних PDF в `pdf_old`, не удалять).
- **Пересборка из кэша** (`rebuild_from_cache`): восстановить `report.xlsx` без запросов
  к API, если отчёт «разъехался».
- Показ статистики папки: уникальных КН, PDF, JSON в кэше.

### 3.5. «Журнал и ошибки»
- Полный лог последнего запуска + история.
- Таблица проблемных объектов (обрыв сети / ошибка API) с КН и причиной.
- Кнопка **«Повторить только проблемные»** → новый прогон по списку этих КН.
- Экспорт лога в файл (для пересылки исполнителю).

### 3.6. «Анализ логов» (отдельное окно просмотра и анализа)
- Каждый прогон пишет структурированный лог `output/logs/run_<дата>.jsonl`
  (поля: время, уровень, КН, статус, сообщение).
- Выбор файла лога из списка прошлых прогонов (или открыть произвольный).
- **Сводка**: всего событий, OK / без сведений / пропущено / обрыв сети / ошибки,
  длительность прогона.
- **Таблица событий** с фильтрами по уровню (INFO/WARN/ERROR) и статусу
  (ok/failed/network/skipped) и поиском по КН/тексту.
- Двойной клик по строке-ошибке → копировать КН (для повторной обработки).
- Экспорт отфильтрованного в CSV.

### 3.7. «Результаты» (сводка)
- Карточки: всего объектов, сформировано, без сведений, пропущено, обрыв сети.
- Кнопки: «Открыть папку результатов», «Открыть report.xlsx».

## 4. Защита от дублей (PDF и строки Excel)

Сейчас при каждом запуске для КН генерируется **новый UUID-PDF**, если объект не
«полностью обработан» — отсюда лишние файлы. Делаем детерминированно:

1. **Индекс `output/cache/kn_index.json`**: `нормализованный КН → {pdf_name, extract_id}`.
2. При обработке КН:
   - если КН уже в индексе и PDF существует → **переиспользуем то же имя** PDF
     (перезапись, не новый файл);
   - в `report.xlsx` строки этого КН **заменяются**, а не добавляются.
3. Итог-инвариант: **1 объект = 1 PDF = 1 блок строк** в отчёте (строк может быть
   несколько, если у объекта несколько прав — это норма).

Дополнительно вкладка «Обслуживание» вычищает исторические дубли от старых прогонов.

## 5. Обработка ошибок и внештатных ситуаций

| Ситуация | Поведение |
|----------|-----------|
| Обрыв/таймаут сети | Ретраи (по умолчанию 5), затем объект **пропускается без записи** → попадёт в «проблемные», повторится при следующем запуске |
| 401 / 402 / 400 | Понятное сообщение (ключ / баланс / нет единиц), без падения |
| `report.xlsx` открыт в Excel | Поймать `PermissionError`, попросить закрыть файл, повторить сохранение |
| Битый PDF/строка (как «row too high») | Уже исправлено в движке; ошибка по объекту логируется, прогон продолжается |
| Падение движка | Перехват исключения в рабочем потоке → красная запись в логе, окно не закрывается |
| Автосохранение | `--save-every` пишет отчёт по ходу — прогресс не теряется при сбое/выключении |

```

│   ├── settings.py          # чтение/запись config.json, settings.json
│   ├── worker.py            # QThread + сигналы (прогресс/лог/итог), отмена
│   ├── engine_bridge.py     # импорт reestro_parser, адаптер событий
│   └── tabs/
│       ├── tab_connection.py

│       ├── tab_maintenance.py

│       └── tab_results.py

Движок берётся из соседней папки `../reestro` (импорт + включение в сборку через
`.spec`). Это позволяет не дублировать отлаженный код парсера.

## 7. Минимальные правки движка (для GUI)

Добавить в `reestro_parser.py`/обёртку, не ломая CLI:
1. `run_batch(params, on_event=..., should_stop=...)` — обработка с колбэками и
   кооперативной отменой (вместо только `print` и `break`).
2. Детерминированное имя PDF по КН + `kn_index.json` (раздел 4).
3. Диапазон объектов `--from N --to M` (сейчас есть только `--limit`).
4. Возврат структурированного итога (числа + список проблемных КН).

CLI остаётся рабочим: GUI и консоль используют одни и те же функции.

## 8. Этапы реализации

| Этап | Содержание | Результат |
|------|------------|-----------|
| 1 | Каркас GUI (PySide6), вкладки-заглушки, настройки/`config.json`, проверка баланса | Окно запускается, подключение проверяется |
| 2 | Обёртка движка `run_batch` с событиями + рабочий поток, вкладка «Пакетная обработка» (прогресс, лог, Старт/Стоп) | Полноценный батч из GUI |
| 3 | Диапазон + одиночные КН + автоимя файла | Вкладки «Одиночный объект», диапазон |
| 4 | Детерминированные PDF + `kn_index.json` (антидубли) + вкладка «Обслуживание» | Дублей нет, очистка из GUI |
| 5 | Журнал/ошибки + «повторить проблемные» + сводка | Полный контроль ошибок |
| 6 | Иконка, `.spec`, сборка PyInstaller, установщик Inno Setup | Готовый `Setup.exe` |

## 9. Принятые решения (значения по умолчанию)

Зафиксированы; при необходимости меняются позже.

1. **GUI**: PySide6 (нативный вид, удобные таблицы; .exe ~80–120 МБ).
2. **Имя отчёта**: авто-шаблон `Отчёт_<входной файл>_<дата>.xlsx` с возможностью
   переопределить вручную.
3. **Нумерация диапазона**: по всем строкам входного файла (как сотрудник видит в
   таблице), чтобы «с 30 по 75» совпадало с номерами строк.
4. **Хранение apiKey**: открытым текстом в `config.json` (простой перенос и правка).
```
