# Инструкция для заказчика (без опыта работы с командной строкой)

**Главное правило:** не нужно вводить пути в чёрное окно вручную.  
Запускайте файлы **двойным щелчком** из папки проекта.

---

## Что было не так на скриншоте

| Ошибка | Почему |
|---|---|
| Окно открыто в `C:\Windows\System32` | Это системная папка Windows, не проект |
| Вставили путь без слова `cd` | Windows думает, что это команда — и ругается |
| Путь `...\reestro_parser_full2\parser` не найден | Папка называется иначе или лежит глубже |

**Правильно:** открыть папку проекта в Проводнике и запускать `.bat` файлы оттуда.

---

## Шаг 0. Найти папку проекта

1. Откройте **Проводник** (жёлтая папка на панели задач).
2. Перейдите туда, куда распаковали архив, например:
   ```
   C:\files_final\files_final\reestro_parser_full2\reestro_parser_full
   ```
   > Путь может отличаться. Ищите папку, где лежат файлы:
   > - `reestro_parser.py`
   > - `1_УСТАНОВКА.bat`
   > - `config.json` или `config.json.example`
   > - папка `TZ`

3. **Как узнать точный путь:** в Проводнике кликните один раз по **адресной строке** вверху — путь выделится. Скопируйте его (Ctrl+C).

4. Для удобства: дважды щёлкните **`0_ОТКРЫТЬ_ПАПКУ_ПРОЕКТА.bat`** — откроется нужная папка.

---

## Шаг 1. Установка (один раз)

1. Дважды щёлкните **`1_УСТАНОВКА.bat`**
2. Дождитесь сообщения «Базовая установка завершена»
3. Нажмите любую клавишу

Появится папка `venv` — это нормально, не удаляйте её.

---

## Шаг 2. Настройка config.json

1. В папке проекта найдите **`config.json`**
2. Откройте **Блокнотом** (правой кнопкой → Открыть с помощью → Блокнот)
3. Вставьте свои `apiKey` и `orgId` от Контур.Реестро
4. Сохраните (Ctrl+S) и закройте

---

## Шаг 3. Положить входной файл

1. Откройте папку **`TZ`**
2. Положите туда файл **`Запрос.xlsx`** (список объектов от заказчика)

---

## Шаг 4. Проверка API

1. Дважды щёлкните **`2_ПРОВЕРКА_БАЛАНСА.bat`**
2. Должно быть: **`balance HTTP 200`**
3. Если ошибка — см. раздел «Частые ошибки» ниже

---

## Шаг 5. Тест на одном объекте

1. Дважды щёлкните **`3_ТЕСТ_1_ОБЪЕКТ.bat`**
2. Дождитесь окончания
3. Откройте папку **`output`**
   - **`report.xlsx`** — должна появиться строка с данными
   - **`pdf`** — должен появиться PDF-файл

Если тест прошёл — можно запускать полный прогон.

---

## Шаг 6. Полное сканирование (весь список)

1. Дважды щёлкните **`4_ПОЛНЫЙ_ПРОГОН.bat`**
2. Нажмите любую клавишу для старта
3. **Не закрывайте окно** — идёт обработка (может занять 20–40 минут на сотни объектов)
4. В окне бегут строки вида `[1/499] ... OK` или `пропуск — уже обработан`

**Можно прервать** (закрыть окно) и **продолжить позже** — снова запустите `4_ПОЛНЫЙ_ПРОГОН.bat`. Уже готовые объекты не обрабатываются повторно.

### Результат

| Папка / файл | Что там |
|---|---|
| `output\report.xlsx` | Сводная таблица по всем объектам |
| `output\pdf\` | PDF на каждый объект |
| `output\cache\json\` | Кэш (не удалять!) |

---

## Шаг 7. Форма собственности (если нужна)

Только после основного прогона:

1. Дважды щёлкните **`5_ФОРМА_СОБСТВЕННОСТИ.bat`**
2. Откроется браузер — решите капчу на сайте Росреестра
3. Следуйте подсказкам в чёрном окне

## Если всё же нужна командная строка

**Не открывайте cmd из System32.** Правильный способ:

1. Откройте папку проекта в **Проводнике**
2. Кликните в **адресную строку** вверху (где путь)
3. Сотрите путь, напишите **`cmd`** и нажмите **Enter**
4. Откроется cmd **уже в нужной папке**

Дальше по очереди:

```cmd
venv\Scripts\activate
python check_balance.py
python run_one_new.py
python reestro_parser.py
```

Перед путём всегда пишите **`cd`** (change directory), если переходите из другой папки:

```cmd
cd /d C:\files_final\files_final\reestro_parser_full2\reestro_parser_full
```

> **`cd /d`** — перейти на другой диск или в другую папку.  
> Без `cd` Windows воспринимает путь как имя программы — та самая ошибка со скриншота.

## Частые ошибки

### «не является внутренней или внешней командой»

Вы вставили путь **без** слова `cd` в начале.  
**Решение:** используйте `.bat` файлы двойным щелчком.

### «Системе не удается найти указанный путь»

Папка названа иначе. Найдите в Проводнике папку с файлом `reestro_parser.py`.

### «No module named requests»

Не выполнена установка. Запустите **`1_УСТАНОВКА.bat`**.

### «ConnectionError» / обрыв соединения

- Выключите VPN
- Нужен интернет с российского IP

### Окно сразу закрывается

Запускайте `.bat` **двойным щелчком из Проводника**, не из ярлыка.  
В конце bat-файлов есть `pause` — окно ждёт нажатия клавиши.

## Советы

1. **Не переносите отдельные файлы** — работайте из одной папки проекта.
2. **Не удаляйте** `output\cache\json\` и `output\report.xlsx`.
3. **Не запускайте** установку каждый день — только `4_ПОЛНЫЙ_ПРОГОН.bat`.
4. Если прервали прогон — просто запустите `4_ПОЛНЫЙ_ПРОГОН.bat` снова.
5. Антивирус может спрашивать разрешение на Python — разрешите для папки проекта.
6. Подробная инструкция: **`ИНСТРУКЦИЯ.md`**
