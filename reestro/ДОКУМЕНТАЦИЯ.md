## 1. Текущее состояние проекта

| Компонент | Статус |
|---|---|
| Основной парсер `reestro_parser.py` | Готов, API `GET /realty/address/v2/objects/{kn}` |
| Генерация PDF (fpdf2) | 2 раздела по образцу ТЗ |
| Генерация Excel `report.xlsx` | 35 колонок, одна строка = одно право |
| JSON-кэш ответов API | `output/cache/json/{кн}.json` |
| Дозапись в report без потери данных | Реализована |
| Форма собственности | Через справочник / кэш / `fetch_rosreestr.py` |
| Пересборка без API | `rebuild_from_cache.py` |
| Тестовый запуск одного объекта | `run_one_new.py` |
| Проверка баланса | `check_balance.py` (бесплатный запрос) |
| Postman-коллекция | `postman/` |

**Требования к окружению:** Python 3.11 или 3.12, Windows / Linux, доступ к `api.kontur.ru` с **российского IP** (без VPN с зарубежным выходом).

**Входной файл `TZ/Запрос.xlsx`** в репозиторий не входит (большой файл заказчика) — его нужно положить в папку `TZ/` перед запуском.

## 2. Что делает проект

Проект автоматически получает **открытые сведения из ЕГРН** по списку объектов недвижимости через API Контур.Реестро и формирует:

- **PDF** — один файл на каждый объект (UUID в имени), два раздела: характеристики + права.
- **report.xlsx** — сводная таблица (35 колонок). Несколько прав на один объект → несколько строк.

**Ограничение API:** открытый метод `address_api_open_data` **не возвращает ФИО собственников**. Доступны: КН, адрес, площадь, назначение, кадастровая стоимость, статус, вид права, номер/дата регистрации, обременения.

**Форма собственности** (Частная / Государственная / …) — только с сайта Росреестра (раздел 10).

### Назначение ключевых файлов

| Файл | Роль |
|---|---|
| `reestro_parser.py` | Ядро. Все остальные скрипты импортируют его функции. |
| `rebuild_from_cache.py` | Пересборка PDF/Excel из кэша, **0 запросов к API**. |
| `fetch_rosreestr.py` | Браузер Chromium: человек решает капчу, скрипт считывает форму. |
| `run_one_new.py` | Находит новые КН из `Запрос.xlsx` и дозаписывает в report. |
| `check_balance.py` | Быстрая проверка авторизации и остатка единиц. |
| `input/ownership_forms.csv` | Ручной справочник форм собственности (`КН;Форма`). |
| `output/cache/json/` | Локальный кэш API. Повторный запуск **не списывает деньги**. |

## 4. Установка Python и зависимостей

### 4.1. Установка интерпретатора Python

**Windows (рекомендуется 3.11 или 3.12):**

1. Скачайте с [python.org/downloads](https://python.org/downloads).
2. При установке **обязательно** отметьте **«Add Python to PATH»**.
3. Проверка в `cmd` или PowerShell:

```cmd
python --version
```

Ожидается: `Python 3.11.x` или `Python 3.12.x`.

**Linux (Ubuntu/Debian):**

```bash
sudo apt update
sudo apt install python3 python3-pip python3-venv
python3 --version
```

**Важно:** используйте один и тот же интерпретатор для создания `venv` и запуска скриптов. Не смешивайте системный Python и Python из Microsoft Store без необходимости.

### 4.2. Виртуальное окружение (рекомендуется)

Изоляция зависимостей от системного Python.

**Windows (cmd):**

```cmd
cd C:\путь\к\parser
python -m venv venv
venv\Scripts\activate
```

**Windows (PowerShell):**

```powershell
cd C:\путь\к\parser
python -m venv venv
.\venv\Scripts\Activate.ps1
```

> Если PowerShell блокирует скрипты:
> `Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser`

**Linux / macOS:**

```bash
cd /путь/к/parser
python3 -m venv venv
source venv/bin/activate
```

После активации в начале строки терминала появится `(venv)`.

**Автоматическая установка (Windows):** запустите `install.bat` или `install.ps1` — скрипт создаст `venv`, установит пакеты и скопирует `config.json.example` → `config.json`.

### 4.3. Установка зависимостей

```bash
pip install -r requirements.txt
```

| Пакет | Назначение |
|---|---|
| `requests` | HTTP-запросы к API Контур.Реестро |
| `openpyxl` | Чтение `Запрос.xlsx`, запись `report.xlsx` |
| `fpdf2` | Генерация PDF |
| `playwright` | Только для `fetch_rosreestr.py` (браузер) |

### 4.4. Chromium для fetch_rosreestr.py (опционально)

Нужен **только** если загружаете форму собственности через браузер:

```bash
python -m playwright install chromium
```

Скачивается ~150 МБ. Для основного парсера не требуется.

### 4.5. Проверка установки

```bash
python -c "import requests, openpyxl, fpdf; print('OK')"
```

Должно вывести `OK`.

## 5. Настройка config.json

Скопируйте шаблон:

```bash
copy config.json.example config.json    # Windows
cp config.json.example config.json      # Linux
```

Заполните:

```json
{
  "baseUrl": "https://api.kontur.ru",
  "apiKey": "ВАШ_API_KEY",
  "orgId": "ВАШ_ORG_ID"

```

Ключи выдаются в личном кабинете Контур.Реестро при подключении API.

**Альтернатива — переменные окружения:**

```cmd
set REESTRO_API_KEY=ваш_ключ
set REESTRO_ORG_ID=ваш_orgId
```

```bash
export REESTRO_API_KEY=ваш_ключ
export REESTRO_ORG_ID=ваш_orgId
```

**Проверка:**

```bash
python check_balance.py
```

Ожидается `balance HTTP 200` и JSON с полем `address_api_open_data`.

## 6. Принцип работы

### Поток данных

```
TZ/Запрос.xlsx (или CSV)
        │
        ▼
reestro_parser.py
        │  для каждой строки:
        │    1) извлечь КН из столбца «ЕГРН»
        │    2) проверить report.xlsx — уже обработан?
        │    3) проверить cache/json/ — есть кэш?
        │    4) если нет → GET api.kontur.ru/.../objects/{kn}
        │    5) сохранить ответ в кэш
        │
        ├──► output/pdf/{uuid}.pdf
        └──► output/report.xlsx (дозапись)
```

### Кэш — экономия платных единиц

При первом успешном запросе ответ сохраняется в `output/cache/json/{кн}.json` (двоеточия в имени заменяются на `_`). При следующем запуске парсер читает кэш и **не обращается к API**.

`rebuild_from_cache.py` работает **только** с кэшем — API не вызывается.

### Критерий «уже обработан»

КН считается обработанным, если в `report.xlsx` есть строка с этим КН, указано имя PDF и файл существует в `output/pdf/`. Такие объекты **пропускаются** при повторном запуске.

### Форма собственности (приоритет)

1. `input/ownership_forms.csv` (ручной справочник)
2. `output/cache/rosreestr/{кн}.json` (кэш Росреестра)
3. Флаг `--fetch-rosreestr` (best-effort, ненадёжно из-за капчи)
4. Рекомендуется: `fetch_rosreestr.py` (браузер + человек)

## 8. Сценарии запуска

### Сценарий 1: Один тестовый объект

```bash
python run_one_new.py
```

Берёт первый КН из `TZ/Запрос.xlsx`, которого нет в `output/report.xlsx`.

```bash
python run_one_new.py --kn 43:40:000184:744
python run_one_new.py --count 5
python run_one_new.py -i input/one_object.csv
```

### Сценарий 2: Полный групповой прогон

```bash
python reestro_parser.py
```

Эквивалент:

```bash
python reestro_parser.py -i TZ/Запрос.xlsx -o output -c config.json --pause 0.4
```

### Сценарий 3: Поэтапная обработка (лимит баланса)

```bash
python reestro_parser.py --limit 50
# Следующий день — первые 50 пропускаются, обрабатываются следующие 50:
python reestro_parser.py --limit 50
```

### Сценарий 4: Произвольный CSV/XLSX (формат Б)

```bash
python reestro_parser.py -i input/objects_example.csv -o output
```

Минимальный CSV (разделитель `;`):

```
кадастровый номер;адрес
43:40:000028:599;Киров г, Октябрьский пр-кт, 9
```

### Сценарий 5: Пересборка без API

После правок шаблона или добавления форм собственности:

```bash
python rebuild_from_cache.py -o output
python rebuild_from_cache.py -o output --ownership-forms input/ownership_forms.csv
python rebuild_from_cache.py -o output --no-pdf   # только Excel
```

### Сценарий 6: Повторная обработка (осторожно!)

```bash
python reestro_parser.py --force --kn ...
```

`--force` удаляет старые строки КН из report и **заново списывает платные единицы API**. Для пересборки без API используйте `rebuild_from_cache.py`.

### Сценарий 7: Форма собственности с Росреестра

```bash
python fetch_rosreestr.py -o output              # КН с правами из report
python fetch_rosreestr.py -o output --all        # все КН
python fetch_rosreestr.py --kn 77:01:0001001:1037
python fetch_rosreestr.py --kn-file kns.csv
python fetch_rosreestr.py -o output --redo       # переспросить известные

python rebuild_from_cache.py -o output
```

### Параметры reestro_parser.py

| Параметр | Описание | По умолчанию |
|---|---|---|
| `-i, --input` | Входной XLSX/CSV | `TZ/Запрос.xlsx` |
| `-o, --output` | Папка результатов | `output` |
| `-c, --config` | config.json | `config.json` |
| `--pause` | Пауза между запросами, сек | `0.4` |
| `--limit N` | Не более N **новых** объектов через API | без лимита |
| `--force` | Переобработать уже готовые КН | выкл |
| `--no-cache` | Не читать/писать JSON-кэш | выкл |
| `--cache-dir` | Путь к кэшу | `{output}/cache/json` |
| `--ownership-forms FILE` | Справочник форм | `input/ownership_forms.csv` |
| `--fetch-rosreestr` | Авто-запрос к lk.rosreestr.ru | выкл |

## 9. Дозапись данных без потери старых

**Ни один скрипт не стирает `report.xlsx` целиком.**

### reestro_parser.py / run_one_new.py

1. Читают существующий `report.xlsx` → `existing_rows`
2. Собирают множество обработанных КН → `processed_kn`
3. Обрабатывают только **новые** КН (не в `processed_kn`)
4. Объединяют: `existing_rows + new_rows`
5. Записывают объединённый файл

PDF создаются с **новым UUID** — старые PDF в `output/pdf/` **не удаляются**.

### rebuild_from_cache.py

1. Пересобирает строки для всех КН из `cache/json/`
2. Сохраняет строки из report, для которых **нет** кэша (пропущенные объекты)
3. Объединяет и записывает

### Что сохранять при переносе на другой ПК

```
output/report.xlsx
output/pdf/
output/cache/json/       ← обязательно для бесплатной пересборки
output/cache/rosreestr/  ← если загружали формы
input/ownership_forms.csv
```

### Чего избегать

- **`--force`** без необходимости — повторное списание API.
- **Удаление `cache/json/`** — потеря возможности бесплатной пересборки.
- **Ручное редактирование report.xlsx** — скрипты определяют готовые КН по этому файлу.

## 10. Получение формы собственности с Росреестра

API Контур.Реестро **не возвращает** форму собственности.

### Способ А: fetch_rosreestr.py (рекомендуется)

```bash
python -m playwright install chromium
python fetch_rosreestr.py -o output --all
```

Откроется браузер. Решите капчу, откройте карточку — скрипт считает «Форму собственности». Сохраняет в:
- `output/cache/rosreestr/{кн}.json`
- `input/ownership_forms.csv`

Затем:

```bash
python rebuild_from_cache.py -o output
```

### Способ Б: Ручной справочник

`input/ownership_forms.csv`:

```
Кадастровый номер;Форма собственности
43:40:000028:599;Государственная федеральная
43:40:000184:742;Частная
```

## 11. Диагностика и частые ошибки

### ConnectionError / Remote end closed connection

Сервер разорвал соединение до ответа. Частые причины:

- VPN с зарубежным IP → выключить или российская локация
- Антивирус / корпоративный прокси → исключить `api.kontur.ru`
- Нестабильная сеть → `--pause 2`

```bash
python check_balance.py
python -c "import requests; [print(h, requests.get('https://'+h, timeout=15, headers={'User-Agent':'Mozilla/5.0'}).status_code) for h in ['api.kontur.ru','lk.rosreestr.ru']]"
```

### HTTP 400 / «нет доступных единиц»

Баланс `address_api_open_data` = 0. Пополните в личном кабинете Контур.Реестро.

### HTTP 401 / 403

Неверный `apiKey` или `orgId` в `config.json`.

### HTTP 404

Объект не найден в ЕГРН по указанному КН.

### Не найден TTF-шрифт (PDF)

**Windows:** должны быть Times New Roman, Arial или Calibri.

**Linux:**

```bash
sudo apt install fonts-liberation fonts-dejavu
```

### Объекты пропускаются без PDF

Нет кадастрового номера в столбце «ЕГРН». Поиск по адресу **отключён**.

```bash

# 2. config.json + TZ/Запрос.xlsx

# 3. Проверка
```
