# Парсер открытых сведений ЕГРН (Контур.Реестро)

> **Для заказчика (без командной строки):** [ИНСТРУКЦИЯ_ДЛЯ_ЗАКАЗЧИКА.md](ИНСТРУКЦИЯ_ДЛЯ_ЗАКАЗЧИКА.md) — запуск через `.bat` файлы.  
> **Подробная инструкция:** [ИНСТРУКЦИЯ.md](ИНСТРУКЦИЯ.md)

Получение **открытых (общедоступных) сведений** об объектах недвижимости из ЕГРН
через API Контур.Реестро (сервис `realty.address-api v2`, боевая площадка).

**Получаемые данные** (без персональных данных собственников):

- кадастровый номер, адрес (ЕГРН + структурный);
- вид и тип объекта, назначение, наименование;
- площадь, этаж / этажность;
- статус объекта (актуально / снято с учёта);
- кадастровая стоимость и дата её определения;
- форма собственности; вид, номер и дата регистрации права;
- ограничения / обременения;
- ФИАС GUID (из входных данных).

> Отчёт носит справочный характер и **не является официальной выпиской** из ЕГРН.

## Правки заказчика (TZ/inst.txt)

1. **«Собственник» (форма собственности).** API Контур.Реестро **не** возвращает
   форму собственности (напр. «Государственная федеральная»). Её источник —
   страница [lk.rosreestr.ru/eservices/real-estate-objects-online](https://lk.rosreestr.ru/eservices/real-estate-objects-online),
   строка «Форма собственности». Значение пишется в столбец
   **«Собственник → Вид собственности»**; если данных нет — «данные отсутствуют».
   Источник формы реализован в три слоя (приоритет сверху вниз):
   - ручной справочник `input/ownership_forms.csv` (или `--ownership-forms FILE`),
     формат `КН;Форма собственности`;
   - кэш `output/cache/rosreestr/{kn}.json` (по одному обращению на КН);
   - авто-запрос к `lk.rosreestr.ru` — флаг `--fetch-rosreestr` (best-effort:
     сервис с капчей, результат нужно проверять; см. ниже);
   - **браузерный загрузчик `fetch_rosreestr.py`** — рекомендуемый способ
     (человек решает капчу, скрипт сам считывает форму; см. ниже).

### Доступ к Росреестру и VPN

`lk.rosreestr.ru` (и платный `api.kontur.ru`) доступны **только с российского IP**.
При VPN с зарубежным выходом оба хоста уходят в таймаут. Проверка:

```bash
python -c "import requests; [print(h, requests.get('https://'+h, timeout=10, headers={'User-Agent':'Mozilla/5.0'}).status_code) for h in ['lk.rosreestr.ru','api.kontur.ru']]"
```

Если таймаут — выключите VPN, выберите российскую локацию или пустите эти хосты
мимо VPN (split-tunnel).

### Браузерный загрузчик формы собственности

```bash
pip install playwright
python -m playwright install chromium

python fetch_rosreestr.py -o output            # КН с правами из report.xlsx
python fetch_rosreestr.py -o output --all      # все КН
python fetch_rosreestr.py --kn 77:01:0001001:1037
```

Откроется браузер: решаете капчу, находите объект, открываете карточку — скрипт
сам считывает строку «Форма собственности», кладёт её в кэш и в
`input/ownership_forms.csv`. Если не распознал — даёт ввести вручную. Известные КН
пропускаются (сессия/капча запоминаются в профиле `output/.pw_profile`).
После загрузки — пересоберите отчёт: `python rebuild_from_cache.py -o output`.

2. **«Доля».** Заполняется только при наличии явной доли (напр. `1/5`); если доля
   не указана (просто «Общая долевая собственность») — ячейка остаётся **пустой**.

### Пересборка без новых платных запросов

По уже обработанным объектам ответ API лежит в `output/cache/json`. Чтобы
применить правки к ним **без обращения к платному API**:

```bash
python rebuild_from_cache.py -o output --ownership-forms input/ownership_forms.csv
```

> `--force` в основном парсере для этого НЕ подходит — он повторно списывает
> платные запросы. `rebuild_from_cache.py` работает только с локальным кэшем
> (запросов к API: 0). Авто-загрузку формы собственности он не делает —
> используйте справочник/кэш.

## Тарифицирование

| Услуга (`code`) | Что расходует | Тариф |
|---|---|---|
| `address_api_open_data` | `GET /objects/{kn}` — данные об объекте | Prepay (единицы) |
| `billing/v1/balance` | Проверка баланса | **Бесплатно** |

Проверить текущий баланс можно в Postman (раздел «0. Служебные») или запросом:
```
GET https://api.kontur.ru/realty/billing/v1/balance
```
**Если баланс `address_api_open_data` = 0 → парсер вернёт ошибку по каждому объекту.**
Пополните счёт в личном кабинете Контур.Реестро.

## 1. Проверка через Postman (перед запуском)

1. Импортируйте в Postman два файла из `postman/`:
   - `Kontur-Reestro-EGRN.postman_collection.json` — коллекция;
   - `Kontur-Reestro-EGRN.postman_environment.json` — переменные (`apiKey`, `orgId` уже заполнены).
2. В правом верхнем углу выберите окружение **«Контур.Реестро — ЕГРН (боевая)»**.
3. Папка **«0. Служебные (авторизация и баланс)»**:
   - **«Проверка авторизации»** — ожидается `200 OK`; `401/403` → ключ неверный.
   - **«Проверка баланса»** — смотрите вкладку **Visualize** для удобного просмотра остатков.
4. Запрос **«3. Информация об объекте по КН»** — замените `cadastralNumber` на нужный.
5. Запрос **«1. Поиск объекта по адресу»** — укажите произвольный адрес.

**Заголовок авторизации** (настроен в коллекции):
```
Authorization: ReestroAuth apiKey={{apiKey}}&portal.orgid={{orgId}}
```

## 2. Запуск парсера

### Установка зависимостей

**Windows (автоматически):** запустите `install.bat` или `install.ps1`.

**Вручную:**

```bash
python -m venv venv
venv\Scripts\activate          # Windows
# source venv/bin/activate     # Linux
pip install -r requirements.txt
copy config.json.example config.json   # заполните apiKey и orgId
python check_balance.py                # ожидается HTTP 200
```

### Входные данные

**Формат A — `TZ/Запрос.xlsx`** (основной, 16+ колонок):

| Столбец | Содержимое |
|---------|-----------|
| Регион | Кировская область |
| Тип/Название населённого пункта, улицы | ФИАС-структура |
| Дом, Корпус, Строение, Помещение | Составные части адреса |
| Доп. Информация | Внешний ID объекта (EXT_NUMBER) |
| **ФИАС** | GUID объекта в ФИАС |
| **ЕГРН** | Кадастровый номер (если известен) |
| Адрес полностью | Собранный текстовый адрес |

**Формат B — CSV/XLSX с произвольными столбцами:**
требует хотя бы одного из: `адрес`, `кадастровый номер` / `ЕГРН`.

### Стратегия определения кадастрового номера

Поиск осуществляется **строго** по кадастровому номеру (из столбца «ЕГРН»).
Если кадастровый номер не заполнен — строка пропускается (помечается как «Без сведений»).
Поиск по адресу отключён для экономии запросов.

### Запуск

```bash
# Из файла Запрос.xlsx (по умолчанию)
python reestro_parser.py

# Произвольный входной файл
python reestro_parser.py -i input/objects_example.csv -o output

# С явным указанием всех параметров
python reestro_parser.py -i TZ/Запрос.xlsx -o output -c config.json --pause 0.5
```

Параметры:

| Параметр | Описание | По умолчанию |
|----------|----------|-------------|
| `-i, --input` | Входной XLSX/CSV | `TZ/Запрос.xlsx` |
| `-o, --output` | Папка для результатов | `output` |
| `-c, --config` | Путь к `config.json` | `config.json` |
| `--pause` | Пауза между запросами, сек | `0.4` |

Ключи можно задать через переменные окружения: `REESTRO_API_KEY`, `REESTRO_ORG_ID`.

### Результат

```
output/
├─ pdf/
│  ├─ 43_40_000028_599.pdf      ← PDF на каждый объект (формат образца ТЗ)
│  └─ ...
└─ report.xlsx                  ← Сводная таблица (35 колонок, одна строка = одно право)
   ├─ Sheet1                    ← Все объекты + права
   └─ Сводка                   ← Статистика (всего / ок / без данных)
```

**Формат PDF** (2 раздела, как в образце ТЗ):
- **Раздел 1** — сведения о характеристиках объекта: кадастровый №, адрес, площадь,
  назначение, кадастровая стоимость, статус, особые отметки.
- **Раздел 2** — сведения о зарегистрированных правах: вид, номер и дата регистрации,
  ограничения/обременения (без ФИО собственников).

**Формат Excel** (`report.xlsx`) — 35 столбцов, идентичных образцу ТЗ:
`EXT_NUMBER | Кадастровый номер | Адрес | Регион | Район | Город | Нас.пункт | Улица | Дом | Корпус | Строение | Помещение | Площадь | Кадастровая стоимость | Стоит на учёте | Дата снятия | Категория земель | Разрешённое использование | … | Собственник (ФИО/тип/доля/право) | Номер выписки | Дата выписки | Отчёт (PDF)`

## 4. Часто задаваемые вопросы

**Q: Почему все объекты получают ошибку «нет доступных единиц»?**
A: Баланс `address_api_open_data` = 0. Пополните в личном кабинете.

**Q: Как проверить баланс?**
A: Запрос «Проверка баланса» в Postman, или:
```bash
curl -H "Authorization: ReestroAuth apiKey=<ключ>&portal.orgid=<orgId>" \
     https://api.kontur.ru/realty/billing/v1/balance
```

**Q: Доступны ли ФИО собственников?**
A: Нет. Открытый метод `address-api` не возвращает персональные данные. Поле
«Собственник» содержит только **вид собственности** (Частная / Государственная / ...).

**Q: Что означает столбец EXT_NUMBER в отчёте?**
A: Это значение из колонки «Доп. Информация» входного файла — внешний идентификатор
объекта из вашей системы.

**Q: Как парсер обрабатывает объекты без кадастрового номера (только ФИАС)?**
A: Парсер строит адресный запрос из столбца «Адрес полностью» или из структурных
колонок (регион, улица, дом и т.д.) и ищет кадастровый номер через
`GET /address?address=...`. Если находит — запрашивает полные сведения по КН.
ФИАС GUID сохраняется в PDF и в Excel.
