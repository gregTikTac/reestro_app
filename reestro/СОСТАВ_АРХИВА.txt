# Состав архива «reestro_parser_full»

Дата актуализации: июнь 2026  
Версия проекта: парсер открытых сведений ЕГРН через Контур.Реестро (address-api v2).

---

## Что НЕ включать в архив

| Папка / файл | Причина |
|---|---|
| `config.json` с реальными ключами | Секреты — заказчик создаёт из `config.json.example` |
| `TZ/ключ API.txt` | Содержит секреты |
| `venv/`, `.venv/` | Виртуальное окружение создаётся на месте (`install.bat`) |
| `output/`, `output_test_first/` | Результаты прошлых прогонов (если не нужны заказчику) |
| `output/.pw_profile/` | Профиль браузера Playwright |
| `.git/` | История git |
| `__pycache__/` | Кэш Python |
| `cursor-chat-export*.json` | Служебные файлы IDE |
| Тестовые скрипты (`test_*.py`, `vvvv.py`, …) | Только для разработки |

## Опционально (если нужна передача наработок)

Если заказчику нужно продолжить с уже обработанными объектами, **добавьте**:

```
output/
├── report.xlsx
├── pdf/
└── cache/
    ├── json/          # Кэш ответов API — обязателен для бесплатной пересборки
    └── rosreestr/     # Кэш форм собственности (если загружались)
```

Без `output/cache/json/` повторная обработка потребует новых платных запросов к API.

---

## Сборка архива

Из папки `reestro` выполните:

```powershell
powershell -ExecutionPolicy Bypass -File prepare_archive.ps1
```

Будет создана папка `dist/reestro_parser_full/` и ZIP-файл `dist/reestro_parser_full.zip`.

---

## Минимальный порядок действий получателя

1. Распаковать архив.
2. Запустить `install.bat` (или `install.ps1`).
3. Заполнить `config.json`.
4. Положить `TZ/Запрос.xlsx`.
5. `python check_balance.py` → HTTP 200.
6. `python run_one_new.py` → один тестовый объект.
7. `python reestro_parser.py` → полный прогон.

Подробности — в `ДОКУМЕНТАЦИЯ.md`.
