## 1. Основные понятия

### Кадастровый номер (КН)

Уникальный идентификатор объекта недвижимости в ЕГРН.  
Формат: `XX:XX:XXXXXXX:XXX`, например `43:40:000184:744`.

Парсер **работает только по КН**. Строки без кадастрового номера пропускаются (в Excel попадают как «без сведений», PDF не создаётся).

---

### API Контур.Реестро

Внешний сервис, откуда парсер получает **открытые сведения ЕГРН**:
адрес, площадь, назначение, кадастровая стоимость, виды прав, обременения и т.д.

**Не возвращает:** ФИО собственников, форму собственности (Частная / Государственная / …).

Адрес API: `https://api.kontur.ru`  
Метод: `GET /realty/address/v2/objects/{кадастровый_номер}`

---

### Платная единица (`address_api_open_data`)

Каждый **новый** успешный запрос сведений по одному КН списывает **1 единицу** с предоплатного баланса.

Проверка баланса — **бесплатная** (`check_balance.py`).

| Действие | Списывает единицы? |
|---|---|
| Первый запрос по КН через API | Да, 1 единица |
| Повторный запуск — КН уже в кэше | Нет |
| Повторный запуск — КН уже в report.xlsx | Нет (пропуск) |
| `rebuild_from_cache.py` | Нет |
| `check_balance.py` | Нет |

---

### config.json

Файл с ключами доступа к API:

```json
{
  "baseUrl": "https://api.kontur.ru",
  "apiKey": "ваш-ключ",
  "orgId": "ваш-org-id"
}
```

Без него парсер не запустится.

---

### Входной файл (`TZ/Запрос.xlsx`)

Таблица со списком объектов для обработки. Основной формат — Excel с колонками:
Регион, адресные поля, **ЕГРН** (кадастровый номер), **Адрес полностью**, **ФИАС**, **Доп. Информация**.

Альтернатива — CSV с колонками `кадастровый номер` и `адрес`.

---

### output/report.xlsx

**Главный результат** — сводная таблица на 35 колонок.  
Одна строка = одно зарегистрированное право. Если у объекта 3 права — будет 3 строки с одним КН.

**Важно:** файл **дозаписывается**, а не перезаписывается. Старые строки сохраняются.

---

### output/pdf/

Папка с PDF-отчётами. Имя файла — UUID, например `213f8689-dff1-4fa4-a9f3-313d0dba19f9.pdf`.  
Один PDF на один объект (не на каждое право).

---

### output/cache/json/

**Кэш ответов API.** После первого успешного запроса по КН ответ сохраняется в файл вида `43_40_000184_744.json`.

При повторном запуске парсер читает кэш и **не тратит платную единицу**.

> **Не удаляйте эту папку** — без неё нельзя бесплатно пересобрать отчёт.

---

### output/cache/rosreestr/

Кэш «Формы собственности» с сайта Росреестра (если использовали `fetch_rosreestr.py`).

---

### input/ownership_forms.csv

Ручной справочник форм собственности:

```
Кадастровый номер;Форма собственности
43:40:000028:599;Государственная федеральная
```

---

### venv (виртуальное окружение)

Изолированная копия Python с установленными библиотеками проекта.  
**Перед каждым запуском** нужно активировать:

```cmd
venv\Scripts\activate
```

В начале строки терминала появится `(venv)`.

---

### Скрипты проекта

| Скрипт | Назначение |
|---|---|
| `check_balance.py` | Проверить ключи и остаток единиц (бесплатно) |
| `run_one_new.py` | Обработать 1 или несколько **новых** КН (тест / дозапись) |
| `reestro_parser.py` | **Основной парсер** — полное или частичное сканирование |
| `fetch_rosreestr.py` | Загрузить форму собственности через браузер |
| `rebuild_from_cache.py` | Пересобрать PDF и Excel из кэша без API |

## 6. Шаг 1. Настройка config.json

Откройте `config.json` в блокноте и заполните:

```json
{
  "baseUrl": "https://api.kontur.ru",
  "apiKey": "ВАШ_КЛЮЧ",
  "orgId": "ВАШ_ORG_ID"
}
```

Сохраните файл.

Проверка:

```cmd
python check_balance.py
```

**Ожидаемый результат:**

```
balance HTTP 200
{"items":[{"id":"...","balance":{"value":86,"unit":"UNIT"},"type":{"code":"address_api_open_data",...
```

| Код | Значение |
|---|---|
| `HTTP 200` | Ключи верны, API доступен |
| `HTTP 401/403` | Неверный apiKey или orgId |
| `ConnectionError` | Проблема с сетью / VPN — нужен российский IP |
| `ModuleNotFoundError` | Не активирован venv или не установлены пакеты |

Найдите в ответе `"code":"address_api_open_data"` и значение `"value"` — это **сколько объектов можно обработать** без пополнения баланса.

## 8. Шаг 3. Тестовый запуск (один объект)

**Зачем:** убедиться, что PDF и Excel формируются правильно, **до** траты всех единиц на полный список.

```cmd
venv\Scripts\activate
python run_one_new.py
```

### Что делает команда

1. Читает `TZ/Запрос.xlsx`
2. Читает `output/report.xlsx` (если есть)
3. Находит **первый КН**, которого ещё нет в report
4. Делает **1 запрос** к API (или берёт из кэша)
5. Создаёт PDF в `output/pdf/`
6. **Дозаписывает** строки в `output/report.xlsx`

### Пример вывода

```
КН уже в report.xlsx: 0
Новых КН в Запрос.xlsx: 499, обрабатываем: 1
  → 43:40:000028:599  (Киров г, Октябрьский пр-кт, 9)

→ Запрос: 43:40:000028:599
  HTTP 200
  ✓ OK | площадь: 45.2 кв.м | прав: 1

Новых обработано: 1  (успешно: 1, ошибок: 0)
report.xlsx → output\report.xlsx
```

### Проверка результата

1. Откройте `output\report.xlsx` — должна появиться строка с КН
2. Откройте PDF из `output\pdf\` — два раздела: характеристики + права
3. Проверьте `output\cache\json\` — должен появиться JSON-файл по этому КН

### Другие варианты теста

```cmd
python run_one_new.py --kn 43:40:000184:744
python run_one_new.py --count 3
python run_one_new.py --kn 43:40:000184:744 --output output
```

### 9.3. Что происходит при полном сканировании (пошагово)

Парсер выполняет следующий цикл **для каждой строки** входного файла:

```
Строка N из Запрос.xlsx
        │
        ▼
   Есть кадастровый номер?
        │
   Нет ─┴─► пропуск (в Excel — «без сведений», PDF не создаётся)
        │
       Да
        │
        ▼
   КН уже в report.xlsx + PDF существует?
        │
   Да ──┴─► пропуск («уже обработан») — API НЕ вызывается
        │
       Нет
        │
        ▼
   Есть файл в output/cache/json/?
        │
   Да ──┴─► данные из кэша — API НЕ вызывается, PDF пересоздаётся
        │
       Нет
        │
        ▼
   GET api.kontur.ru/.../objects/{КН}
        │
        ├─ HTTP 200 ──► сохранить в кэш → PDF → строки в report
        │
        └─ HTTP 404/400/401 ──► PDF «не найден» → строка с ошибкой в report
        │
        ▼
   Пауза 0.4 сек → следующая строка
        │
        ▼
   В конце: объединить старые + новые строки → записать report.xlsx
```

### 9.5. Сколько времени займёт

Примерная оценка:
- ~0.4 сек пауза + ~1–3 сек запрос API = **~2–4 сек на объект**
- 500 объектов ≈ **15–35 минут**

Если часть уже в кэше или report — быстрее (пропуск без API).

---

### 9.6. Поэтапное сканирование (если единиц меньше, чем объектов)

Если на балансе 86 единиц, а объектов 499:

```cmd
python reestro_parser.py --limit 86
```

После пополнения баланса — снова:

```cmd
python reestro_parser.py --limit 100
```

Парсер **автоматически пропустит** уже обработанные КН и возьмёт следующие новые.

Можно запускать **сколько угодно раз** — данные **дозаписываются**, не стираются.

### 9.11. Чего НЕ делать при полном сканировании

| Действие | Почему плохо |
|---|---|
| `python reestro_parser.py --force` | Повторно списывает единицы API за уже готовые КН |
| Удалять `output/cache/json/` | Потеряете возможность бесплатной пересборки |
| Удалять `output/report.xlsx` | Парсер «забудет», что уже обработано → повторные запросы |
| Редактировать report вручную | Может нарушить логику «уже обработан» |
| Запуск без `(venv)` | `ModuleNotFoundError: requests` |

## 10. Шаг 5. Форма собственности (опционально)

API не возвращает форму собственности. Если нужна в столбце «Собственник → Вид собственности»:

### Способ А — браузер (рекомендуется)

```cmd
python -m playwright install chromium
python fetch_rosreestr.py -o output --all
```

1. Откроется браузер Chromium
2. Зайдите на lk.rosreestr.ru
3. Решите капчу, откройте карточку объекта
4. Нажмите Enter в консоли — скрипт считает «Форму собственности»
5. Повторяется для каждого КН

Затем обновите отчёт:

```cmd
python rebuild_from_cache.py -o output
```

### Способ Б — ручной справочник

Заполните `input/ownership_forms.csv`:

```
Кадастровый номер;Форма собственности
43:40:000028:599;Государственная федеральная
```

```cmd
python rebuild_from_cache.py -o output --ownership-forms input\ownership_forms.csv
```

```cmd
python rebuild_from_cache.py -o output
```

```cmd
python rebuild_from_cache.py -o output --no-pdf
```

### check_balance.py — проверка баланса

## 14. Типичные ситуации и решения

### «ModuleNotFoundError: No module named 'requests'»

```cmd
venv\Scripts\activate
pip install -r requirements.txt
```

### «ConnectionError» / «Remote end closed connection»

- Выключите VPN или выберите российский сервер
- Проверьте антивирус / корпоративный прокси
- Увеличьте паузу: `--pause 2`

### «HTTP 400 / нет доступных единиц»

Пополните баланс `address_api_open_data` в личном кабинете Контур.Реестро.

### «Все КН уже присутствуют в report.xlsx»

Полное сканирование завершено. Для новых объектов — добавьте их в `Запрос.xlsx` и запустите снова.

### Нужно переобработать один объект заново

```cmd
python reestro_parser.py --force
```

⚠ Списывает платную единицу. Для обновления шаблона без API используйте `rebuild_from_cache.py`.

### Перенос на другой компьютер

Скопируйте всю папку `output/` (report, pdf, cache/json).  
На новом ПК — установите venv, config.json, запустите `rebuild_from_cache.py`.
